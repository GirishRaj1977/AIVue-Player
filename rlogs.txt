For an IPTV player, the most useful logs are the ones that help you diagnose playback failures, portal issues, EPG problems, crashes, and user complaints without creating huge log files.

1. Application Lifecycle

Log startup and shutdown events.

[APP] Version 1.4.2 started
[APP] OS: Windows 11
[APP] User data path: C:\Users\...
[APP] Shutdown requested

Useful for:

Crash investigation
Version-specific bugs
Startup failures
2. Playlist / Portal Loading
Xtream
[XTREAM] Login attempt
[XTREAM] Server: example.com
[XTREAM] Authentication successful
[XTREAM] Loaded 1250 channels
[XTREAM] Loaded 4200 movies
M3U
[M3U] Downloading playlist
[M3U] Playlist size: 18000 entries
[M3U] Parse completed
Stalker
[STALKER] Handshake successful
[STALKER] Profile loaded
[STALKER] Categories fetched: 42
[STALKER] Channels fetched: 1250

Useful for:

Login issues
Parsing failures
Portal incompatibilities
3. Playback Logs (Most Important)

When a user says "channel not working", this is what you'll check.

[PLAYER] Play request
[PLAYER] Channel: ESPN HD
[PLAYER] URL: http://...
[PLAYER] Engine: MPV

[PLAYER] Stream opened
[PLAYER] Video codec: h264
[PLAYER] Audio codec: aac

[PLAYER] Playback started

Failures:

[PLAYER ERROR] HTTP 403
[PLAYER ERROR] HTTP 404
[PLAYER ERROR] Connection timeout
[PLAYER ERROR] Decoder initialization failed
4. MPV Logs

Keep relevant MPV events.

[MPV] start-file
[MPV] file-loaded
[MPV] playback-restart
[MPV] end-file

Errors:

[MPV ERROR] HTTP error 403
[MPV ERROR] Invalid data found
[MPV ERROR] Network timeout

Avoid logging every MPV debug line in production.

5. Stream Statistics

Useful for buffering complaints.

[STATS]
Bitrate: 8.5 Mbps
Dropped frames: 12
Cache: 15s
Resolution: 1920x1080

Log only periodically (every 30–60 seconds).

6. EPG Logs
[EPG] Download started
[EPG] Source loaded
[EPG] Parsed 12500 programmes
[EPG] Refresh completed

Failures:

[EPG ERROR] XML parse failed
[EPG ERROR] Download timeout
7. Recording Logs (if supported)
[RECORDING] Started
[RECORDING] File: movie.ts
[RECORDING] Stopped
[RECORDING] Duration: 01:25:12
8. User Actions

Don't overdo this.

Useful:

[USER] Changed channel
[USER] Opened guide
[USER] Added favourite
[USER] Search: HBO

Not useful:

Mouse moved
Key pressed
Window resized
9. Database Logs
[DB] Opened database
[DB] Migration 12 applied
[DB] Favourite saved

Errors:

[DB ERROR] Constraint failed
10. Crash Logs

Capture:

Timestamp
App version
Stack trace
Current channel
Current portal type
Current URL

Example:

[CRASH]
Version: 1.4.2
Channel: ESPN HD
Portal: Xtream
Stack:
...
11. Update Logs
[UPDATE] Checking for updates
[UPDATE] New version available
[UPDATE] Download complete
What NOT to Log

Avoid storing:

User passwords
Xtream passwords
MAC addresses (mask them)
Full tokens
Full stream URLs in release builds

Instead:

http://server.com/...play_token=*****
MAC: 00:1A:79:**:**:F6
Recommended Log Levels
INFO    Normal operations
WARN    Recoverable issues
ERROR   Failed operations
DEBUG   Development builds only

For your Electron + MPV IPTV player, I'd create separate rotating log files:

logs/
 ├─ app.log
 ├─ player.log
 ├─ portal.log
 ├─ epg.log
 ├─ crash.log

and automatically delete logs older than 7–14 days. This gives enough information to troubleshoot user issues while keeping disk usage under control.
